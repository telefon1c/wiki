# Провайдеры и линии

Справочник **«Звонки и сообщения» → «Провайдеры»** используется для определения внешней линии, через которую был совершен звонок.

Это позволяет различать звонки разных операторов, учетных записей и телефонных номеров компании. Вместе со справочником **Каналы привлечения** данные о линии позволяют анализировать источник интереса клиента и эффективность рекламных номеров.

## Как определяется провайдер

При загрузке истории звонков АТС передает в 1С технический идентификатор провайдера и DID.

Технический идентификатор — это ID endpoint SIP или IAX на вашей АТС Asterisk. В 1С он хранится в поле **Код** справочника **Провайдеры**.

DID — это номер линии, на которую поступил входящий звонок. В зависимости от настроек оператора это может быть телефонный номер или логин учетной записи провайдера.

Сначала 1С определяет провайдера по полю **Код**. Если подходящий элемент отсутствует, он создается автоматически. Изначально его наименование совпадает с техническим идентификатором, а при последующей синхронизации заменяется на понятное наименование провайдера с АТС.

Для входящих звонков 1С дополнительно ищет DID в табличной части **Номера телефонов** справочника **Провайдеры**. Если DID найден, в поле **Линия** журнала звонков записывается провайдер, которому принадлежит этот номер.

!!!info Приоритет определения линии
DID имеет более высокий приоритет, чем **Код** провайдера. Если провайдер был определен по коду, но DID указан в карточке другого провайдера, в журнал звонков будет записан провайдер, найденный по DID.
!!!

## Назначение полей справочника «Провайдеры»

### Наименование

Название линии, которое будет отображаться в журнале звонков, например:

- Основной номер;
- Отдел продаж;
- Megafon — Москва;
- Megafon — Санкт-Петербург.

### Код

Технический идентификатор провайдера на АТС — ID endpoint SIP или IAX в Asterisk.

Для провайдеров, созданных автоматически, поле заполняется техническим идентификатором провайдера из истории звонков. При ручном создании элемента поле **Код** заполнять не требуется: оно останется пустым автоматически.

### Номера телефонов

Табличная часть содержит DID, по которым входящий звонок можно связать с конкретной линией. В ней могут быть указаны:

- телефонные номера компании;
- логины учетных записей провайдера;
- другие значения DID, передаваемые АТС в поле **Номер линии**.

Значение должно точно совпадать со значением **Номер линии** в данных звонка.

## Несколько учетных записей на одном адресе

Сложности могут возникнуть, если настроено несколько учетных записей одного оператора с одинаковыми IP-адресом или доменом и портом, но разными логинами.

Например:

| Провайдер | Адрес | Порт | Логин |
|---|---|---:|---|
| Megafon — Москва | sip.megapbx.ru | 5060 | 74951234567 |
| Megafon — Санкт-Петербург | sip.megapbx.ru | 5060 | 78121234567 |

При входящем звонке Asterisk может определить провайдера только по IP-адресу и порту. Поскольку эти параметры совпадают, звонок может быть связан с первым найденным техническим идентификатором провайдера.

В результате в поле **Линия** журнала звонков будет указан неверный провайдер.

Чтобы настроить точное сопоставление:

1. В разделе **«Звонки и сообщения»** откройте справочник **«Провайдеры»**.
2. Создайте отдельный элемент для каждой учетной записи.
3. Укажите понятное наименование, например **Megafon — Москва**.
4. В табличной части **Номера телефонов** укажите логин учетной записи или телефонный номер, передаваемый АТС как DID.
5. Поле **Код** оставьте без изменений. У созданного вручную элемента оно будет пустым.
6. Запишите элемент.
7. Повторите действия для остальных учетных записей.

После настройки новые входящие звонки будут сопоставляться исходя из значения DID, а не только по техническому идентификатору провайдера.

!!!warning
Если разные учетные записи передают одинаковый DID, определить их средствами справочника невозможно. В этом случае необходимо скорректировать настройки провайдера, входящих маршрутов или SIP-заголовков на АТС.
!!!

## Один логин и несколько DID

Если одна учетная запись обслуживает несколько входящих номеров, создавать отдельного провайдера для каждого DID обычно не требуется.

Создайте один элемент справочника и добавьте все номера в его табличную часть:

| Номер |
|---|
| 74951234567 |
| 74957654321 |
| 74959876543 |

Все звонки с одним из этих DID будут связаны с этим провайдером.

Если разные DID должны отображаться как самостоятельные линии, для каждого DID можно создать отдельный элемент справочника и указать в нем соответствующий номер.

## Провайдеры и каналы привлечения

Справочники **Провайдеры** и **Каналы привлечения** решают разные задачи:

| Объект | На какой вопрос отвечает |
|---|---|
| Провайдер или линия | Через какую учетную запись или endpoint АТС поступил звонок? |
| Канал привлечения | На какой номер позвонил клиент и с каким источником интереса связан этот номер? |

Например, несколько рекламных номеров могут обслуживаться одной учетной записью оператора:

| DID | Провайдер | Канал привлечения |
|---|---|---|
| 74951234567 | Megafon | Контекстная реклама |
| 74957654321 | Megafon | Реклама на сайте |
| 74959876543 | Megafon | Наружная реклама |

В этом примере провайдер у всех звонков одинаковый, но источники привлечения различаются.

## Справочник «Каналы привлечения»

Справочник **«Звонки и сообщения» → «Каналы привлечения»** предназначен для классификации входящих звонков по DID. Он позволяет анализировать эффективность:

- рекламных кампаний;
- номеров на разных сайтах;
- региональных телефонных номеров;
- номеров отдельных подразделений;
- различных источников интереса клиентов.

Канал привлечения записывается в запись журнала звонков и используется в отчете **Распределение звонков по каналам привлечения**.

### Автоматическое заполнение

Для каждого входящего звонка 1С получает DID и ищет в справочнике **Каналы привлечения** элемент с таким номером телефона.

Если элемент с таким номером отсутствует, он создается автоматически:

- поле **Номер телефона** заполняется значением DID;
- поле **Наименование** остается пустым.

Таким образом, справочник постепенно наполняется номерами по мере загрузки входящих звонков.

!!!info
Каналы привлечения определяются только для входящих звонков. Для исходящих и внутренних звонков канал привлечения автоматически не заполняется.
!!!

### Настройка наименований

После автоматического создания элементов необходимо присвоить номерам понятные наименования:

1. В разделе **«Звонки и сообщения»** откройте справочник **«Каналы привлечения»**.
2. Откройте элемент с нужным номером телефона.
3. Заполните поле **Наименование**.
4. Запишите изменения.

Примеры наименований:

| Номер телефона | Наименование |
|---|---|
| 74951234567 | Контекстная реклама |
| 74957654321 | Реклама на сайте |
| 74959876543 | Наружная реклама |
| 78001234567 | Бесплатная линия |
| 78121234567 | Санкт-Петербург |

Поле **Номер телефона** должно содержать DID в том формате, в котором его передает АТС. Не следует изменять номер без необходимости: если при следующем звонке АТС передаст другое значение, 1С создаст для него новый элемент справочника.

### Объединение нескольких номеров

Нескольким номерам можно присвоить одинаковое наименование:

| Номер телефона | Наименование |
|---|---|
| 74951234567 | Контекстная реклама |
| 78121234567 | Контекстная реклама |
| 78001234567 | Контекстная реклама |

В отчете данные элементов с одинаковым наименованием суммируются. Это позволяет объединить несколько региональных или подменных номеров в один аналитический канал. При этом сами номера остаются отдельными элементами справочника.

### Заполнение ранее загруженных звонков

Новые входящие звонки получают канал привлечения автоматически.

Чтобы заполнить или пересчитать канал привлечения для ранее загруженных звонков:

1. Откройте список **Каналы привлечения**.
2. Нажмите **Заполнить каналы привлечения**.
3. Укажите дату, начиная с которой необходимо обработать журнал звонков.
4. Дождитесь завершения фоновой операции.

Будут обработаны все входящие звонки начиная с указанной даты. Для каждого звонка канал привлечения будет повторно определен исходя из значения DID.

Если требуется только переименовать существующий канал, повторное заполнение журнала не требуется: записи звонков уже содержат ссылку на элемент справочника и будут отображать новое наименование.

### Использование в отчетах

Справочник используется в отчете **Распределение звонков по каналам привлечения**. Отчет позволяет определить:

- сколько звонков поступило по каждому рекламному номеру;
- какие источники дают больше обращений;
- как изменяется количество обращений по дням, неделям и месяцам;
- по каким каналам поступают пропущенные звонки;
- эффективность отдельных рекламных кампаний.

Если у нескольких элементов одинаковое наименование, их показатели в отчете объединяются.

## Как узнать передаваемый DID

Значение для табличной части справочника **Провайдеры** должно совпадать с полем **Номер линии** в журнале звонков.

Для проверки:

1. Выполните тестовый входящий звонок.
2. Откройте его в журнале звонков.
3. Посмотрите значение **Номер линии**.
4. Укажите это значение в табличной части нужного провайдера.
5. Выполните новый тестовый звонок.

Настройка провайдера применяется к новым звонкам. Уже загруженные записи истории автоматически не перераспределяются.

## Важные ограничения

- Сопоставление провайдера по табличной части выполняется только для входящих звонков.
- Значения DID сравниваются как строки, без нормализации.
- Формат номера должен совпадать: `74951234567` и `+7 495 123-45-67` считаются разными значениями.
- Один DID не следует указывать у нескольких провайдеров — результат поиска станет неоднозначным.
- Табличная часть **Номера телефонов** справочника **Провайдеры** автоматически с АТС не заполняется.
- Для исходящих звонков провайдер определяется по техническому идентификатору, полученному от АТС.
- Если DID не передан, автоматически определить канал привлечения невозможно.

## Результат настройки

После правильного заполнения справочников:

- новые входящие звонки получают корректное значение поля **Линия**;
- учетные записи одного оператора можно различать даже при одинаковом IP-адресе и порте;
- несколько DID можно объединить под одним понятным наименованием провайдера;
- для каждого DID можно указать отдельный источник интереса клиента;
- отчет по каналам привлечения показывает эффективность рекламных номеров и кампаний.
